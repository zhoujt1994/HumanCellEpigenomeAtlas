# Data provenance & how to regenerate

Reproduction starts from a small set of **provided inputs**; everything else is a pipeline
intermediate that is either **generated by a notebook in this book** or, for the heavy
upstream products, **assumed to exist with its generating tool documented here**.

## Provided inputs (assume these exist)
| Input | What | Generated by |
|---|---|---|
| **single-cell mcds** (`*.mcds`) / raw `mcad` | per-cell CG/CH methylation matrices | **ALLCools** (from allc) |
| **DMR ds** (`DMR/…`, RegionDS) | differentially methylated regions | ALLCools DMR calling |
| **loop bedpe + contact cools** (`{ct}.loop.bedpe`, `{ct}.loop_summit.bedpe`, `Q/Q2/E/E2/T/T2.cool`) | per-cell-type chromatin loops & imputed contacts | **scHiCluster** |
| **cluster annotation** (`5kCG100k3C_summary.csv.gz`, `subtype_meta.tsv`, `L1color.tsv`) | cell metadata + major/subtype labels & colors | clustering pipeline |
| **cell×binpair npz** (`raw/*.npz`, `embedding/…/*.npz`) | single-cell 3C feature matrices (for HiC clustering) | **scHiCluster** |
| **`*.boundary.h5ad`** | aggregated TAD-boundary probabilities | **scHiCluster** (domain). *No figure uses single-cell domain files.* |

The **mC embedding + integration** (LSI→PCA→biosketch-downsampled anchor integration→Leiden)
is a per-major-type **template** run across all types; only one template example needs to be
included. This is **not yet added** — the notebooks currently read the resulting
`*_embed.h5ad` (assume exist; generated by ALLCools clustering in
`{ENTEX_ROOT}/clustering/merged`). See `NONCONFIDENT.md`.

# Required input files (per notebook)

For each notebook this lists the files it **reads**, with two flags to help you decide what must be (re)computed versus what already exists:

- **`RAW`** — the filename is not written by any notebook in this collection, so it is a *raw external input* you must supply (single-cell allc/cool, references, external datasets, hand-made metadata).

- **`DERIVED`** — the same filename is written by some notebook here, so it is a *pipeline intermediate*. You either already have it or it is produced by an earlier (pipeline) notebook — see the run-order notes in each figure chapter.

Path expressions keep their original variables (`{indir}`=`{ENTEX_ROOT}/`, `{outdir}`, `{cooldir}`, …). Paths built purely from variables or inside shell (`!`/`os.system`) cells may not appear — notably the `bedtools`/`awk` steps and `pysam` tabix allc reads.

```{admonition} Please review
:class: tip
Mark each `RAW` file as *have* or *need to compute*. The `DERIVED` files are produced by the pipelines in this repo, so they mostly reduce to their own `RAW` inputs.
```
